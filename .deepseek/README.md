# OpenCode 連携システム

## 概要

このディレクトリは、OpenCode（DeepSeek V4 Flash OpenCode）との連携用です。
タスクをファイルとして配置し、OpenCodeが読んで実装する仕組みです。

## ディレクトリ構造

```
.deepseek/
├── tasks/
│   ├── inbox/          # 新規タスクを置く場所（あなたが作成）
│   ├── in_progress/    # OpenCodeが作業中のタスク
│   └── completed/      # 完了したタスク
├── questions/
│   ├── pending/        # OpenCodeからの質問（回答待ち）
│   └── answered/       # 回答済みの質問
├── context/            # プロジェクトコンテキスト情報
└── README.md           # このファイル
```

## 使い方

### 1. タスクを作成

`tasks/inbox/` に新しいタスクファイルを作成します。

**ファイル名の形式:** `YYYYMMDD_HHMM_タスク名.md`

例: `20260601_1730_プラグインシステム実装.md`

### 2. タスクの書き方

```markdown
# タスク: [タスク名]

## 目的
[何を実現したいか]

## 要件
- [ ] 要件1
- [ ] 要件2
- [ ] 要件3

## 制約
- 既存コードへの影響を最小限に
- コメントは最小限
- null安全性を遵守

## 参考
- 関連ファイル: `lib/xxx/yyy.dart`
- 参考ドキュメント: `AGENTS.md`

## 備考
[その他の情報]
```

### 3. OpenCodeが作業

1. OpenCodeが `inbox/` をチェック
2. タスクを読んで実装
3. 完了したら `completed/` に移動
4. 実装内容をタスクファイルに追記

### 4. 確認

`completed/` のタスクファイルを確認して、実装内容をレビューします。

## コンテキスト情報

`context/` ディレクトリには、プロジェクト全体の情報を配置します。

- `project_overview.md` - プロジェクト概要
- `coding_rules.md` - コーディング規約
- `architecture.md` - アーキテクチャ

## ワークフロー例

```
[あなた]
  ↓ タスク作成
tasks/inbox/20260601_1730_機能追加.md
  ↓ OpenCodeが発見・作業開始
tasks/in_progress/20260601_1730_機能追加.md
  ↓ 不明点があれば質問状を作成
questions/pending/20260601_1730_機能追加_質問.md
  ↓ [あなた] 質問ファイルに回答を記入 → answered/に移動
questions/answered/20260601_1730_機能追加_質問.md
  ↓ OpenCodeが回答を読んで作業再開
tasks/in_progress/20260601_1730_機能追加.md
  ↓ 実装完了
tasks/completed/20260601_1730_機能追加.md
  ↓ レビュー
[あなた] コードレビュー & マージ
```

## Tips

### 効率的なタスク作成

- **具体的に書く**: 曖昧な指示は避ける
- **参考を示す**: 既存コードや設計書を参照
- **制約を明記**: やってはいけないことを書く

### タスクの優先度

ファイル名の先頭に優先度を付けることもできます:

- `P1_20260601_1730_緊急バグ修正.md` (最優先)
- `P2_20260601_1730_機能追加.md` (通常)
- `P3_20260601_1730_リファクタリング.md` (低優先)

## 質問状のルール（OpenCode向け）

### 質問状を出すタイミング

- タスクの要件が矛盾している
- 設計の方向性を確認したい
- 複数の実装方法があり選択が必要
- 参照先ファイルが見つからない

### 質問状の作り方

1. `questions/pending/` に質問ファイルを作成
2. ファイル名: `YYYYMMDD_HHMM_[タスク名]_質問.md`
3. テンプレート: `questions/pending/TEMPLATE.md` を参照
4. 作業中のタスクは `in_progress/` に置いたまま待機
5. `answered/` に移動した質問ファイルを確認して作業再開

### 回答の確認方法

`questions/answered/` にファイルが移動していたら回答済み。
ファイル内の「回答欄」を読んで作業を再開すること。
