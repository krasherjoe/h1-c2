## 目的

プラグイン管理画面から個別プラグインを有効/無効切り替えできるようにする。
無効プラグインのルート・メニュー・DashboardSectionをダッシュボードから除外する。

## 実装仕様

### `plugin_state_service.dart`（新規）

```dart
class PluginStateService {
  static const _kPrefix = 'plugin_enabled_';

  Future<bool> isEnabled(String pluginId) async {
    final prefs = await SharedPreferences.getInstance();
    return prefs.getBool('$_kPrefix$pluginId') ?? true; // デフォルトは有効
  }

  Future<void> setEnabled(String pluginId, bool enabled) async {
    final prefs = await SharedPreferences.getInstance();
    await prefs.setBool('$_kPrefix$pluginId', enabled);
  }

  Future<Map<String, bool>> loadAll(List<String> pluginIds) async {
    final prefs = await SharedPreferences.getInstance();
    return {
      for (final id in pluginIds)
        id: prefs.getBool('$_kPrefix$id') ?? true,
    };
  }
}
```

### `PluginRegistry` への追加

```dart
// 無効プラグインIDのSet
final Set<String> _disabledPlugins = {};

// 有効/無効切り替え（再起動不要）
void setEnabled(String pluginId, bool enabled) {
  if (enabled) {
    _disabledPlugins.remove(pluginId);
  } else {
    _disabledPlugins.add(pluginId);
  }
}

bool isEnabled(String pluginId) => !_disabledPlugins.contains(pluginId);

// 既存メソッドを有効プラグインのみ返すように変更
List<H1Plugin> get allPlugins =>
  _plugins.values.toList(); // 全て（管理画面用）

List<H1Plugin> get activePlugins =>
  _plugins.values.where((p) => isEnabled(p.id)).toList(); // 有効のみ

Map<String, WidgetBuilder> getAllRoutes() {
  final routes = <String, WidgetBuilder>{};
  for (final plugin in activePlugins) {
    routes.addAll(plugin.getRoutes());
  }
  return routes;
}

List<MenuItem> getAllMenuItems() =>
  activePlugins.expand((p) => p.getMenuItems()).toList();
```

**注意:** `getAllRoutes()` はアプリ起動時に `main.dart` で一度だけ呼ばれるため、
ルートの動的更新はできない。メニューとDashboardSectionは動的に反映される。

### `main.dart` への変更

起動時に保存済みの無効状態を復元:

```dart
// registry.register(...) が全部終わった後に追加
final stateService = PluginStateService();
final states = await stateService.loadAll(registry.allPlugins.map((p) => p.id).toList());
for (final entry in states.entries) {
  if (!entry.value) {
    registry.setEnabled(entry.key, false);
  }
}
```

### `plugin_management_screen.dart` への変更

各プラグインカードに `Switch` トグルを追加:

```dart
// _buildPluginCard の trailing に追加
trailing: Switch(
  value: _registry.isEnabled(plugin.id),
  onChanged: _isToggleable(plugin.id)
    ? (val) async {
        _registry.setEnabled(plugin.id, val);
        await PluginStateService().setEnabled(plugin.id, val);
        setState(() {});
      }
    : null, // 無効化できないプラグインはグレーアウト
),
```

**トグル不可にするプラグイン（コアに近いもの）:**
```dart
bool _isToggleable(String pluginId) {
  const nonToggleable = {
    'com.h1.plugin.settings',
    'com.h1.core',
  };
  return !nonToggleable.contains(pluginId);
}
```

**依存関係チェック:**
無効化しようとするプラグインに依存しているプラグインが有効な場合は警告ダイアログを表示:

```dart
// registry._findDependentPlugins を活用
final dependents = registry.allPlugins
  .where((p) => p.dependencies.contains(pluginId) && registry.isEnabled(p.id))
  .map((p) => p.name)
  .toList();
if (dependents.isNotEmpty) {
  // 警告ダイアログ表示
  showDialog(context, ...) // 「〇〇が依存しています。それでも無効にしますか？」
}
```

## 制約

- `com.h1.plugin.settings` は常時有効（トグル不可）
- ルートの動的追加・削除は不要（再起動で反映される旨をUIで表示）
- `_findDependentPlugins` は既存メソッドを使う
- null安全性・コメント最小限
